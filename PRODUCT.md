# Product

## Register

product

## Platform

web

## Users

A single user (you) tracking your own budget and sports bets on a bigger screen than the mobile app allows. The context is deliberate desktop use: reviewing longer bet history, adjusting reward card rates, digging into trend and reporting views that don't fit comfortably on a phone. No auth, no multi-tenant concerns — this is a personal instrument, not a product for strangers.

## Product Purpose

Budgerr Web is the full-mirror desktop client for Budgerr, a personal budgeting app that treats sports betting as a first-class budget category rather than an afterthought. Every feature the FastAPI backend exposes gets a page here. Success looks like opening the app and immediately trusting the numbers: what's left in the betting allowance, which bets are still pending, whether a reward card is being underused — without having to think about whether the data is current or correct.

## Positioning

The one place that treats betting like any other budget line — tracked, capped, and reported on with the same rigor as groceries or rent, on the screen where you actually want to review it.

## Brand Personality

Calm, clear, trustworthy. This is money and betting data — the interface should never feel anxious, gamified, or hypey. No countdown-timer urgency, no celebratory confetti on a win. Confidence comes from density done well and numbers that are easy to scan, not from decoration. Borrow from modern fintech (Mercury, Ramp, Apple Card): clean structure, confident type, restrained color used deliberately rather than everywhere.

## Anti-references

Avoid the generic AI-SaaS look: no gradient text, no cream/sand/parchment neutral backgrounds, no cookie-cutter identical card grids, no tiny uppercase tracked eyebrows above every section, no hero-metric-plus-gradient-accent template. Also avoid sportsbook/gambling-app energy (neon, odds-ticker urgency, celebratory win animations) — this is a budgeting tool that happens to track bets, not a betting app.

## Design Principles

- Numbers are the interface. Every screen exists to answer a specific question (what's left, what's pending, which card) at a glance — legibility and scan-ability beat visual flourish.
- Calm confidence, not gamification. Betting data gets the same sober treatment as any other budget category; no urgency cues borrowed from sportsbooks.
- Desktop is for depth. This mirror exists because some views (longer history, rate management, reporting) genuinely need the room a phone doesn't have — lean into density where the mobile app can't.
- Trust through consistency. Same data model and patterns as the mobile app; a user switching between the two should never be confused about what a number means.

## Accessibility & Inclusion

No formal WCAG target — single-user personal tool. Still hold to standard contrast and readable-type care as baseline practice, since the app displays financial figures that need to be read correctly at a glance.
